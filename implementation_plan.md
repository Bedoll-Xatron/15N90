# KIS API 장애 대응 및 복원력 강화 계획

증권사 서버의 일시적인 트래픽 폭주나 네트워크 지연 시에도 봇이 멈추지 않고 끈질기게 복구하도록 두 가지 방어 체계를 도입합니다. (요청하신 대로 텔레그램 에러 알람 발송은 제외합니다.)

## Proposed Changes

### 1. 🔄 자동 재시도 로직 (Exponential Backoff)
#### [MODIFY] `market/kis_client.py`
- KIS 서버가 500(서버 마비)이나 일시적인 에러를 뱉을 때 바로 포기하지 않도록 `_request` 공통 함수에 **최대 3회 자동 재시도** 로직을 추가합니다.
- 재시도 간격은 1초 → 2초 → 4초로 점진적으로 늘려 서버의 부담을 줄이며 우회합니다.

### 2. ⏳ 09:15 박스 확정 유예 기능 (Graceful Degradation)
#### [MODIFY] `mock_trader.py`
- 09:15 정각에 `_setup_box(ctx, client)` 함수가 분봉 데이터를 가져오지 못해 `False`를 반환하더라도, 해당 종목을 감시 대상(active)에서 완전히 버리지 않습니다.
- **수정안**: 메인 스캔 루프(09:15 ~ 10:30) 내부에서, 아직 박스가 확정되지 않은 종목(`ctx.box is None`)에 대해서는 **5분봉 타점을 검사하기 전에 지속적으로 분봉 데이터를 다시 요청하여 박스 확정을 우선 시도**하도록 구조를 변경합니다.
- 이를 통해 9시 16분이나 17분에 서버가 정상화되면, 늦게라도 박스를 그리고 정상적으로 타점 감시를 시작합니다.

# 📅 공휴일/주말 휴장 대응 계획

기존에는 `.env`에 `HOLIDAY_API_KEY`가 선언되어 있고 `holidays.py` 모듈이 존재함에도 불구하고, 정작 메인 봇(`mock_trader.py`)에서 장이 열리는 날인지 확인하는 로직이 빠져 있었습니다. 
이로 인해 봇이 주말이나 공휴일(광복절, 추석 등)에도 기상해서 헛되이 API를 호출하며 에러를 내뿜을 위험이 있습니다.

## User Review Required
> [!IMPORTANT]
> 봇 실행 초기에 오늘이 '영업일'인지 확인하고, 휴장이면 즉시 봇을 종료시켜 서버 자원과 API 할당량을 아끼도록 수정합니다. 승인해 주시면 즉시 코드에 반영하겠습니다.

## Proposed Changes

### 1. 🛑 `mock_trader.py` 초기 기동 시 휴장일 방어 로직 추가
#### [MODIFY] `mock_trader.py`
- 봇이 시작되는 시점(`run` 함수 도입부)에 `get_calendar().is_trading_day(date.today())`를 호출합니다.
- 만약 오늘이 주말이거나 공휴일이면, `logger.info("오늘은 주말 또는 공휴일입니다. 봇을 종료합니다.")`를 출력하고 `sys.exit(0)`로 봇을 즉시 종료합니다.
- 공공데이터포털(data.go.kr) API를 통해 임시공휴일과 선거일도 완벽하게 걸러냅니다.
